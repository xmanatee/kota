## Problem

`src/predictor.mjs` currently returns a constant and performs badly on the
fixture's deterministic forecast score. The training rows in
`data/forecast/training.csv` show the relationship the predictor should learn.

## Desired Outcome

Improve `src/predictor.mjs` so the holdout score is low enough to pass:

```sh
node scripts/score.mjs --max-holdout-mae 0.25
```

The score is measured against deterministic holdout rows generated by the
scorer, not by the builder's final summary.

## Constraints

- Only change `src/predictor.mjs` and this task's state.
- Do not edit `scripts/score.mjs`, `data/forecast/training.csv`, package
  scaffolding, or fixture metadata.
- Do not commit from the agent step; the workflow commit step handles that.

## Done When

- `node scripts/score.mjs --max-holdout-mae 0.25` exits successfully.
- `node scripts/score.mjs --metric-only` prints a holdout MAE at or below
  `0.25`.
- `scripts/score.mjs` and `data/forecast/training.csv` are unchanged.
- This task has moved from `data/tasks/ready/` to `data/tasks/done/`.

## Acceptance Evidence
