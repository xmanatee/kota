# Docs

This directory holds durable, repo-wide reference documentation.

- Keep docs high-level, current, and useful beyond one implementation run.
- Do not keep changelogs, migration notes, inventories, or task queues here.
- Prefer local `AGENTS.md` files for scoped conventions.
- If code, types, or tests can make behavior clear, improve them instead of
  adding explanatory prose.
- Module-specific operator setup belongs in the module's own `AGENTS.md`, not here.
- Design documents and historical audit records do not belong here.
